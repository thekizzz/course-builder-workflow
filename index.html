📋 Các Phase chi tiết

0
PLAN + SCAN MANDATORY /ck:plan
4 câu hỏi user (dạy gì, mục đích skill, ưu tiên, rủi ro) → scan-modules.py → decision tree chọn pipeline → /ck:plan lưu plans/.
1
CONCAT
concat-module.py gom SRT/DOCX/XLSX/text-PDF theo module. Auto-skip image-PDF. Output {slug}.txt.
2
DISTILL Gemini
Pipeline A (≤10 modules, trong Claude) hoặc Pipeline B (batch distill-batch.py via gemini-2.5-flash). Script idempotent → resume được. Mỗi module 5-10 phút, 30 modules ~3-4h background.
2.5
OCR image-PDFs Gemini API
ocr-pdfs-api.py --priority cho templates/swipes. Subagent ghi TEMP staging (không có quyền ghi ~/.claude/). Pipeline C Claude = OPT-IN only.
2.6
FINALIZE OCR
finalize-ocr.py copy TEMP → data/ocr/ + rebuild AUTO-OCR-INDEX trong 00-index.md.
2.7
COVERAGE AUDIT MANDATORY GATE
coverage-audit.py --threshold 30 --fail-on-flag. Flag folder > threshold% unprocessed → FAIL. Phải quay lại OCR hoặc note skip rõ ràng.
3
GENERATE SKILL.md MANDATORY /ck:code-review
Điền template → /ck:code-review. Nếu >500 dòng → /ck:simplify. Tối ưu lazy-loading: 1 keyword → 1 file.

🚢 PHASE 4 — SHIP (7 steps bắt buộc)

❌ KHÔNG có override
User nói "skip đi cho nhanh" → Reply: "Course-builder quy định Phase 4 là mandatory gate. Chạy đủ 7 steps mới ship. Không có override."
4.1
Coverage Gate MANDATORY
coverage-audit.py --fail-on-flag = exit 0. Nếu fail → quay về 2.5 OCR thêm.
4.2
/ck:scenario MANDATORY
Generate eval cases theo 12 dimensions. Append vào evals/evals.json. Gate: ≥ 10 cases (knowledge + trigger + refusal + injection + ethics).
4.3
/ck:skill-creator (validate + package) MANDATORY
quick_validate.py → "Skill is valid!" + package_skill.py tạo zip. Trước khi package: xóa file rác (.rar, .zip cũ, tmp_*, backup).
4.4
Parallel Eval Grading RECOMMENDED
prep-evals-parallel.py + N subagents background. Nhanh hơn sequential 5-10×. Accuracy target ≥ 85%.
4.5
/ck:security STRIDE audit MANDATORY
Target ≥ 80%. Nếu thấp → apply 3 fix mặc định (data integrity + file access + ethical limits) → re-audit → re-package.
4.6
/ck:autoresearch (description) CONDITIONAL
Chỉ chạy khi description < 250 chars HOẶC trigger test < 80%. Skip nếu đã pushy đủ.
4.7
/ck:watzup final checklist MANDATORY
Review all changes. Confirm ready-to-ship. Warning còn → fix → re-run.
✅ Composite Score
final = accuracy × 0.8 + security × 0.2 · Target ≥ 85%. Dưới ngưỡng → loop 4.2 → 4.5.

🍰 Tóm tắt nôm na — Làm bánh vs Build Skill

PhaseNhư làm bánhBuild skill là
0. Plan Coi công thức, đếm trứng bột, chọn lò nướng phù hợp. Scan folder, đếm SRT/PDF, chọn Pipeline A/B, lập plan bằng /ck:plan.
1. Concat Gom nguyên liệu vào tô — bột riêng, đường riêng. Gom SRT/DOCX/XLSX của từng module vào 1 file .txt.
2. Distill Nấu chảy bơ, đánh bột mịn — Gemini làm phụ bếp rẻ tiền. Gemini distill 30 modules/batch → file data/*.md. Claude không đụng vào.
2.5. OCR Đọc công thức chép tay → đánh máy lại. Ảnh scan = Gemini OCR. Image-PDF qua Gemini OCR. Claude chỉ xài khi Gemini hết quota + user đồng ý.
2.6. Finalize Chuyển bánh từ nồi phụ sang khay chính. Copy TEMP → skill folder + cập nhật index.
2.7. Coverage Đếm lại: thiếu bơ, đường không? Thiếu thì quay lại chợ. Audit folder nào OCR chưa đủ → quay lại làm thêm.
3. Generate Tạo hình bánh, trang trí — đây là lúc Claude làm chính bếp. Claude viết SKILL.md + /ck:code-review kiểm tra.
4. Ship Kiểm tra chất lượng trước khi giao khách — 7 bước không được bỏ. 7 CK gates: scenario → validate → package → security → watzup. Thiếu 1 = không giao.

💡 Tại sao không được skip Phase 4?

Nếu skip…Hậu quả thực tế
Skip /ck:scenarioKhông test edge case → user gửi prompt injection là skill lộ hết data/
Skip /ck:skill-creatorKhông có zip → không share được → skill "trong đầu" mình thôi
Skip /ck:securitySCOPE LOCK có lỗ hổng → skill dạy viết MLM, phishing được
Skip /ck:watzupKhông có final check → sót file rác (như vụ .rar lọt vào zip neville-copywriting)

🎯 TL;DR cho lần build khóa sau

  1. Gemini làm text + OCR. Claude KHÔNG đọc raw PDF/image.
  2. Pipeline C Claude OCR = OPT-IN. Luôn hỏi user trước.
  3. Phase 2.7 coverage audit là gate cứng — fail là không đi tiếp.
  4. Phase 4 có 7 steps, 5 MANDATORY, 1 RECOMMENDED, 1 CONDITIONAL.
  5. Xóa file rác trước khi package_skill.py (không zip .rar/.zip cũ vào).
  6. Composite ≥ 85% mới được ship. Không lower ngưỡng.
  7. User nói "skip" → TỪ CHỐI theo script có sẵn trong SKILL.md.
